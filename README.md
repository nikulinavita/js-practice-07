# Практическая работа №7: События в JS. Kanban-доска

**Стек технологий:** HTML5, CSS3, JavaScript.

**Структура проекта:** - `index.html` — основная разметка интерактивной витрины услуг.
- `style.css` — готовые стили оформления (с поддержкой тёмной темы).
- `script.js` — логика приложения (добавление, удаление, фильтрация, валидация).
- `README.md` — документация проекта и ответы на контрольные вопросы.
- `screenshots/` — папка со скриншотами работающего приложения (добавление карточек, счетчик, тёмная тема, избранное).

## Описание проекта
В данной работе реализована интерактивная Kanban-доска для управления задачами. Проект демонстрирует навыки работы с моделью событий браузера, механизмами всплытия и погружения, а также оптимизацию кода через делегирование событий.

## Ответы на вопросы для интервью

### 1. Что такое делегирование событий и зачем оно нужно?
Делегирование — это прием, при котором один обработчик событий назначается на родительский элемент (в нашем случае `#board`) для управления событиями его дочерних элементов (кнопок в карточках). 
**Зачем:** Это экономит память (не нужно вешать сотни обработчиков) и позволяет автоматически обрабатывать события на новых элементах, которые добавляются в DOM динамически.

### 2. Чем event.target отличается от event.currentTarget?
- `event.target` — это "цель", элемент, на который пользователь нажал физически (например, иконка `✕` внутри кнопки).
- `event.currentTarget` — это элемент, на который мы повесили сам обработчик события (в нашем коде это всегда `#board`).

### 3. Что делает stopPropagation() и когда его использовать?
Метод `stopPropagation()` останавливает всплытие события вверх по дереву DOM. 
**Пример:** Мы использовали его на кнопках действий в карточке, чтобы при клике на кнопку "Удалить" событие не шло дальше к самой карточке и не вызывало её "выделение" (toggle class `selected`).

### 4. Чем addEventListener лучше onclick?
- Позволяет навешивать несколько разных обработчиков на одно и то же событие.
- Поддерживает дополнительные настройки (опции), такие как `{ once: true }` или `{ capture: true }`.
- Не перезаписывает другие обработчики, в отличие от свойства `element.onclick = ...`.

### 5. Что такое { once: true } в addEventListener?
Это опция, которая указывает браузеру, что обработчик должен сработать ровно один раз, после чего он будет автоматически удален. Мы использовали это для закрытия приветственного баннера.

### 6. Почему removeEventListener требует именованную функцию?
Для удаления обработчика браузеру нужна ссылка на ту же самую область памяти, где хранится функция. Анонимные (стрелочные) функции каждый раз создаются заново, поэтому у браузера нет на них постоянной ссылки, и он не сможет их опознать для удаления.
